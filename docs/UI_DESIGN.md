# 别花乐 (BieHuaLe) — 前端设计文档

> **版本**：v2.0  
> **日期**：2026-07-19  
> **状态**：**Clarity Teal（清晰青绿）** — 推翻 v1.x「雾青软萌」；对齐主流记账 App 的「想用」体验  
> **配套**：[PRD.md](./PRD.md) §7 · [STRUCTURE.md](./STRUCTURE.md) · [AGENTS.md](../AGENTS.md)  
> **废止**：Mist Teal Soft Ledger（v1.2 / v0.5.0 视觉取向）不再作为实现标准

---

## 0. 为什么推翻上一代视觉

v0.4–0.5 在品牌色与去 Card 上有进步，但落到真机后出现「没欲望打开」：

| 现状观感（用户反馈） | 根因 |
|----------------------|------|
| 整页淡青雾底 + 灰紫选中 | 对比弱，像未完成皮肤，不像主力工具 |
| 类别只有文字格子 | 主流记账（鲨鱼 / 随手记等）靠**色圆 + 图标**建立扫视与情绪 |
| 金额区灰条 + 保存按钮发灰 | 主行动不像可点；「记一笔」动力被视觉压扁 |
| 软萌 / 轻卡通圆角 | 亲切有余、专业不足；不像 2025–26 主流消费级金融工具 |

本版目标：**打开就想记一笔、记完觉得利落**。参考主流产品共性（高对比中性底、强主 CTA、类别图标矩阵、首屏一个英雄数字），而不是再堆雾感与软萌。

与 PRD 冲突时：**产品规则以 PRD 为准，视觉以本文 v2 为准**（可覆盖 PRD §7.1 过时表述）。

## 1. 产品调性

### 1.1 一句话

**利落的私人记账工具**：三秒记完、一眼看清——本地、安静、但**看起来值得天天用**。

### 1.2 气质关键词

| 要 | 不要 |
|----|------|
| 清晰、利落、有行动欲 | 雾面发灰、软萌卡通、儿童贴纸感 |
| 中性底 + 饱和点缀 | 整页淡青铺满、淡紫选中发脏 |
| 类别「看得见」 | 纯文字类别墙 |
| 主按钮「叫你点」 | 禁用灰 masquerading 为主按钮 |
| 现代消费级 App | 银行冷仪表盘 / 加密霓虹 / 系统设置脸 |
| 克制动效 | 毛玻璃炫技、吉祥物连环画 |

### 1.3 欲望测试（比品牌测试更优先）

任选冷启动进记账页，5 秒内应同时成立：

1. 知道现在在记**支出/收入/转账**哪一种  
2. 金额区是全页最醒目的东西  
3. 类别可以靠**色 + 图标**扫完，不必读完每个字  
4. **保存**在可记状态下是高对比主色块，不是灰条  

账单页 2 秒内读出「本月已花」金额。任一失败 → 视觉不合格。

### 1.4 品牌信号（次级）

去掉底栏后仍应保留：

1.  saturation 足够的青绿主色（不是壁纸乱色）  
2. 金额数字专属字体与层级  
3. 关键空态可出现「别花乐」一次（勿每屏刷品牌）

## 2. 视觉方向（选定方案）

**方向名：Clarity Teal（清晰青绿）**

对齐主流记账 / 轻金融 App 的结构逻辑（2025–26）：**角色清晰 → 英雄指标 → 下一动作**，装饰服从功能。

| 维度 | 规范 |
|------|------|
| **表面** | 浅色默认：近白 `#F7F8F8` 底 + 纯白内容层；**禁止**整页雾青渐变铺满。深色：近黑炭 `#121414` + 抬升面 `#1C1F1E` |
| **主色** | 品牌青绿保持 PRD 系，但 **UI 上用更醒目的工作色**：Light `brand.primary = #0B7A6A`（比旧 `#006C5C` 略提亮饱和）；强调色仅用于焦点，不作大底 |
| **语义色** | 支出 / 收入 / 转账高对比；支出红、收入绿、转账蓝——只染金额与模式指示，不染整页 |
| **结构** | 少重阴影 Card；分区用留白 + 细分割。列表行无 Card。需要成块点击时用圆角浅底（见 §3） |
| **圆角** | **现代中等圆**：字段 / 键帽 `12`；按钮 / Sheet `16`；大面板 `20`。废止 Soft Ledger 的 14/20/28「鼓圆」取向 |
| **图标** | 导航 / 工具用线性精致套件（可继续 `BhlIcons` 但需**加粗对比、去粉嫩**）。**类别必须带图标**（见 §3.5） |
| **Dynamic Color** | 默认关；可设置开启 |
| **冷启动** | 默认 **记账 Tab**（高频动作）；账单仍是「看」的主屏 |

### 2.1 明确避开

- 雾青整页底 + 淡紫底栏选中（上一代失败脸）  
- 紫白 / 紫靛 AI 渐变；奶油底 + 陶土衬线；报纸密排  
- emoji 当类别图标；吉祥物连环画  
- 大面积 iOS 毛玻璃；霓虹描边；多层阴影贴纸墙  
- 保存按钮在「可保存」时仍呈现灰残废态  

### 2.2 与上一代的决裂清单

| Soft Ledger (废止) | Clarity Teal (现行) |
|--------------------|---------------------|
| 纸感雾底 + 噪点主氛围 | 近白底，氛围最多顶区极淡 |
| 软萌鼓圆 | 中等现代圆角 |
| 类别纯文字 | **色圆 + 图标 + 短标签** |
| 弱对比主按钮 | 可点即实心主色 |
| 空态大淡色线稿主角 | 空态短文案 + 小图标 + 强 CTA |

## 3. 设计 Token

### 3.1 色彩

#### 品牌

| Token | Light | Dark | 用途 |
|-------|-------|------|------|
| `brand.primary` | `#0B7A6A` | `#3DDCBE` | 主按钮、选中 Tab、关键控件 |
| `brand.onPrimary` | `#FFFFFF` | `#003730` | 主色上的字 |
| `brand.container` | `#D5F3EC` | `#094A40` | 选中类别底、轻强调（面积宜小） |
| `brand.ink` | `#0A1614` | `#E8F2F0` | 主文字 |

#### 语义（不随壁纸漂移）

| Token | Light | Dark | 用途 |
|-------|-------|------|------|
| `semantic.expense` | `#E11D48` | `#FB7185` | 支出金额、支出模式 |
| `semantic.income` | `#059669` | `#34D399` | 收入金额、收入模式 |
| `semantic.transfer` | `#2563EB` | `#60A5FA` | 转账 |

#### 表面

| Token | Light | Dark |
|-------|-------|------|
| `surface.base` | `#F7F8F8` | `#121414` |
| `surface.elevated` | `#FFFFFF` | `#1C1F1E` |
| `surface.muted` | `#EEF1F0` | `#262A29` |
| `stroke.subtle` | `#E2E6E5` | `#2E3332` |
| `text.secondary` | `#5C6B66` | `#9AA8A4` |
| `text.disabled` | `#A8B2AF` | `#5A6360` |

### 3.2 字体

| 角色 | 字体 | 说明 |
|------|------|------|
| UI / 正文 | `Noto Sans SC` | 清晰现代；默认界面主体 |
| 品牌词（少用） | `Noto Serif SC` | 仅「别花乐」等品牌瞬间 |
| 金额 | `IBM Plex Mono` tabular | 英雄数字与列表金额 |

| 样式 | 字号 | 字重 | 用途 |
|------|------|------|------|
| `money.hero` | 44–52 | SemiBold | 本月已花 / 记账金额舞台 |
| `money.row` | 17–18 | Medium | 流水行 |
| `title.screen` | 20–22 | Sans SemiBold | 页标题（现代感优先于衬线） |
| `body` | 15–16 | Regular | 正文 |
| `caption` | 12–13 | Regular | 辅助 |

### 3.3 间距与圆角

| Token | 值 | 用途 |
|-------|-----|------|
| `space.xs` | 4 | 紧贴 |
| `space.sm` | 8 | 行内 |
| `space.md` | 16 | 屏边、模块 |
| `space.lg` | 24 | 大区块 |
| `space.xl` | 32 | 呼吸 |
| `radius.sm` | **12** | 字段、键帽、类别格 |
| `radius.md` | **16** | 主按钮、Sheet |
| `radius.lg` | **20** | 大面板 |

**区块分离（v0.6.9）：** 账单/记账表单/设置分组等模块用 `surface.elevated`（实现上 `surfaceContainerLow`）+ `radius.md` 圆角「岛」面板包一层；块与块间距 `space.md`（大块可用 `space.lg`）。**禁止**行级 Card 墙、多阴影叠层；流水行仍无 Card（整块「最近」等外包一层面板即可）。共享组件：`SectionPanel`。

主按钮高度 **52**；可保存时 **实心 `brand.primary` + `onPrimary` 字**；不可保存时才用 `surface.muted` + `text.disabled`。

底栏：白 / elevated 面 + 顶部分割线；选中 = 主色图标 + 字，整块 Tab 铺浅 `brand.container` 圆角方底（**对比要够**，禁止灰紫脏色 / 禁止只高亮图标小胶囊）。

### 3.4 导航与工具图标

- 线性、笔画清晰、对比足够；可演进现有 `BhlIcons`，但需视觉加粗、去「淡线稿」感。  
- 顶栏图标 `onSurface` / `onSurfaceVariant`；激活用 `brand.primary`。  
- 筛选已生效：图标旁 **6dp 主色圆点**。

### 3.5 类别图标（硬性）

主流记账的核心爽点之一。规范：

- 每个类别：**直径 40–44dp 色圆底** + **中心单色/简线图标**（24dp）+ 下方短标签。  
- 色圆来自类别 `colorHex`（过淡则自动加深到可读）。  
- 选中：色圆描边 `brand.primary` 2dp 或外圈光环；未选中无灰框监狱感。  
- 内置类别必须有图标资源映射（餐饮碗筷、交通车、购物袋…）；缺图标时用首字圆形，但**发布前应补齐内置集**。  
- **禁止**记账页出现「纯文字 4 列墙」。

## 4. 动效

少而准，服务「利落」。实现统一走 `AppMotion`（`ui/theme/Motion.kt`），曲线一律 `FastOutSlowIn`：

| 动效 | 规格 | 位置 |
|------|------|------|
| 模式切换 | 指示条位移动画 **180ms**；字重/选中色即时 | 支出/收入/转账 |
| 金额变化 | Hero 入场 fade+上移 4dp（仅首次）；**禁止**键盘逐键 pulse | 账单/详情 Hero；键盘仅键帽 `graphicsLayer` 按下反馈 |
| 列表变更 | item fade/placement **180ms**（`animateItem`） | 账单最近 / 全部流水软删 |
| 确认面板 | fade + 轻 scale **200ms** | LedgerConfirm |
| 键盘按键 | 按下 scale **70ms** | MoneyKeypad |
| Sheet | 系统容器变换约 **200ms**，无 overshoot | 筛选 / 长按 |
| 导航转场 | 主 Tab 互切 fade **120ms**；二级页 push/pop slide+fade **200ms**（覆盖 Navigation 默认 700ms） | `NavHost` / `AppMotion.NavTabMs` · `NavPushMs` |

禁止：整页闪白、循环闪烁、强弹跳、大面积 blur；禁止列表包一层永远 `visible=true` 的假 AnimatedVisibility。

## 5. 导航与信息架构（视觉层）

三 Tab：**账单 | 记账 | 设置**（顺序不变）。

1. **冷启动 = 记账**（行动优先）。  
2. 底栏常驻；子页时父 Tab 高亮。  
3. 交互深度（沿用 v0.4.2，不回退）：

| 作业 | 深度 | 归属 |
|------|------|------|
| 看本月 | 0-tap | 账单 |
| 记一笔 | 金额 + 类别 + 保存 | 记账 |
| 查/搜/筛 | ≤2 步 | 全部流水 |
| 改/删 | ≤2（长按或详情） | 列表 / 详情 |

## 6. 分屏构图

### 6.1 记账 Tab（冷启动主屏）—— 欲望核心

竖向节奏：

1. **模式**：支出 | 收入 | 转账 — 文字 Tab + 粗指示条（语义色），不要三块灰按钮  
2. **金额舞台**：近白或极浅底上的巨大 `money.hero`；符号用语义色，数字用 `brand.ink`；**不要灰胶囊条把数字关进笼子**  
3. **快速记账下拉**（仅支出、非编辑）：选中即落账；菜单项「类别 · 说明 + 金额」；空则进设置管理  
4. **类别下拉**：与账户同构的 `LedgerField` + 菜单；菜单项为色圆+图标+名称（§3.5）；无类别时点进类别管理  
5. **账户 / 时间 / 说明**：单行浅底字段（`surface.muted`），整行可点；无账户时文案引导去管理  
6. **保存**：全宽主色实心；贴在键盘上方  
7. **键盘**：浅分割键帽、圆角 `radius.sm`；按下有轻反馈  

快速记账 / 类别 / 账户时间 / 说明 / 转账区整体包一层 elevated 面板；**金额舞台 + 模式**仍在底面上，不关进笼子。保存与键盘贴底，不进面板。

编辑态：标题「编辑账单」；模式锁定。新建保存后留页连记。

### 6.2 账单 Tab—— 一个数字说完

**首屏只保留：**

1. 月份切换（清晰 chevron + 居中月份）  
2. 标签「本月已花」+ **英雄金额**（高对比）  
3. 一行：收入 · 结余（次级）  
4. 一行：总资产（次级、可点看分户；**字号不得大于 Hero**；与月份无关）  
5. 可选极细比例轨  

**不要**：首屏饼图/折线/筛选墙；不要大面积雾底把数字洗掉；不要把总资产做成第二 Hero。

向下：花在哪（饼图可点进全部流水+类别）→ 趋势 → 最近 5 笔 → 查看全部。  
各块用 elevated 面板分离（Hero / 花在哪 / 趋势 / 最近），底色近灰上可见「岛」。  
空态：短文案 + 小图标 + **「去记账」主按钮样式**（不是淡紫弱链）。

### 6.3 全部流水 / 详情

- 搜/筛在此；筛选即时生效；Sheet 现代圆角壳。  
- 列表：色点 + 类别名；金额右对齐。长按编辑/删除。月分区头保留；列表底可用 elevated 面板 + 加大分区间距（避免双滚动感）。  
- 详情：金额区与信息区分别/合并用面板；编辑实心主按钮；删除危险文字降权。

### 6.4 设置与管理

- 分组：标题下内容包 `SectionPanel`（白底圆角岛）；组间距 `space.md`；行高舒适；右 chevron。  
- 关于：「别花乐」+「完全本地 · 不联网」。  
- 账户/类别/快速记账/回收站：列表外包轻量面板；行点击 = 编辑；⋮ 仅归档等次要操作；列表预览必须带色圆图标。

## 7. 组件约定

| 组件 | 约定 |
|------|------|
| 主按钮 | 实心 primary，高 52，`radius.md`；可点态禁止发灰 |
| 次按钮 | TextButton；描边仅必要时用 `stroke.subtle` |
| 字段 | 浅底 `surface.muted`，无粗灰 Outlined 监狱框；聚焦底边/轻环 `brand.primary` |
| Sheet | elevated 面，顶角 `radius.md`，短拖条 |
| 确认框 | 圆角面板；危险操作用文字危险色 |
| 类别下拉项 | §3.5 色圆+图标+名称 |
| Snackbar | 深底短句（「已保存」） |
| 流水行 | 无 Card；分割线可选 |
| 区块面板 | elevated 浅底 + `radius.md`；模块级分离；禁止行级 Card 墙与多阴影叠层 |

共享实现（`ui/common/`，v0.6.6+）：

| 组件 | 文件 | 职责 |
|------|------|------|
| `SubScreenScaffold` / `BackTopAppBar` | `SubScreenScaffold.kt` | 二级页透明 Scaffold + 返回顶栏；可选 actions / FAB / snackbar |
| `SectionPanel` | `SectionPanel.kt` | 区块 elevated 圆角面板（账单/记账表单/设置组/管理列表） |
| `CategoryIconCircle` | `CategoryIconCircle.kt` | 色圆 + `CategoryIconMap` 图标（管理行、下拉项） |
| `LoadingState` | `States.kt` | 居中「加载中…」 |
| `EmptyState` | `States.kt` | 空态 + 可选 `PrimaryButton` CTA |
| `ListSectionHeader` | `ListSectionHeader.kt` | 列表分区小标题 |
| `PrimaryButton` | `PrimaryButton.kt` | 高 52、实心 primary |
| `ManageListRow` | `ManageListRow.kt` | 色圆 + 标题/副标题 + trailing（账户/类别/快速记账） |
| `SearchBar` / `FilterBottomSheet` | 同名文件 | 全部流水搜筛（自 bill 迁入） |
| `LedgerField` / `LedgerSheet` / `LedgerConfirm` | `LedgerComponents.kt` | 字段 / Sheet / 确认 |
| `TransactionRow` | `TransactionRow.kt` | 流水行（小色点，非管理色圆） |
| `CategoryIconMap` / `BhlIcons` | 对应文件 | 图标键与矢量集 |

主 Tab（账单/记账/设置）不套 `SubScreenScaffold`，仅复用分区头、色圆、主按钮等零件。
